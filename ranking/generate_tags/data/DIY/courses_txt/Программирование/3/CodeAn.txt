План курса “Анализ кода и
безопасная разработка”

Авторы: Екатерина Шеметова, Алексей Москвин
Краткое описание курса
На курсе будут рассмотрены как теоретические основы, связанные с современными
вопросами кибербезопасности и анализа кода, в частности, такие темы, как symbolic &
concolic execution, CFL-reachability & summary-based анализ, верификация программ, Model
Checking, Theorem Proving, верификация в реальном времени и тестирование, так и
практические аспекты, а именно обнаружение и защита от различного типа уязвимостей,
таких как уязвимости веб-приложений, приложений на языках общего назначения (Java, PHP,
С#); анализ безопасности смарт-контрактов и исполняемых файлов. Также будут
рассмотрены вопросы разработки новых подходов по обнаружению и защите приложений.
Темы лекций
1. Введение в предметную область. Информационная безопасность, основные
направления, уязвимости и их типы.
2. Поиск уязвимостей в приложениях с помощью паттернов и правил: CodeQL, SemGrep.
3. Графы в анализе программ. AST, CFG, definition-use граф, Code Property Graph.
4. Фреймворк CFL-reachability. Ограничения на множества.
5. Анализ потока данных с помощью монотонного фреймворка.
6. Анализ кода с помощью Datalog. Основы Datalog.
7. Анализ кода с помощью Datalog. Формулировка анализа программы в терминах
Datalog. Пример: анализ указателей Андерсена.
8. Межпроцедурный анализ. Чувствительность к контекстам вызова. Doop, BDD.
9. Символьное исполнение. Forward-, backward- символьное исполнение.
10. Символьное исполнение. Ограничения на пути. SMT-решатели.
11. Фаззинг. AFL.

12. Формальные методы в безопасности приложений. Infer. Верификация смарт-
контрактов.

13. Model Checking. Темпоральная логика. LTL-формулы. Автоматы Бюхи.
14. Theorem Proving. Z3 theorem prover.
15. Динамический анализ программ.
Дополнительные темы, рассматриваемые на практиках
1. Уязвимости приложений: серверная часть – LFI, чтение файлов, command injection,
object injection. Обнаружение с помощью символьного исполнения, паттернов,
Datalog, Code Property графов.
2. Безопасность запросов к базам данных. Внедрение SQL-кода, слепая SQL инъекция,
NoSQL инъекция, защита от атак типа внедрение SQL-кода. Обнаружение с помощью
анализа потока данных.
3. Уязвимости приложений: клиентская часть. Межсайтовый скриптинг (XSS),
межсайтовая подделка запроса (CSRF). Обнаужение с помощью паттеров,
динамического анализа.
4. Введение в бинарные уязвимости: переполнение буфера, перполнение кучи,
форматировка строки.
